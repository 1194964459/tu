# 物流数字化解决方案智能门户平台——核心功能梳理（产品/客户视角）

> 目标：把《土组-门户网站方案.docx》中“做什么、为谁做、如何形成闭环”的核心功能抽取出来，形成后续实现可直接落地的功能逻辑与优先级。

## 1. 产品一句话与边界

- 一句话：面向物流行业的“AI 对话驱动”解决方案选型与验证平台，把“需求采集 → 方案推荐 → 在线试用 → 反馈回流”打通成闭环。
- 不做（当前边界内不重点做）：纯内容型应用商店（只提供截图/文档/人工咨询），只做流量曝光不做试用验证与反馈闭环。

## 2. 核心用户与成功标准

### 2.1 企业购买方（需求方）

- 核心任务（JTBD）
  - 我想把“说不清的需求”变成可选型的指标，快速找到适配方案并验证效果，降低试错与采购风险。
- 成功标准（可量化）
  - 从进入平台到拿到可比对的 1-3 套方案 ≤ 15 分钟（含对话收集）。
  - 完成一次在线试用并产出试用报告 ≤ 1 小时（用样例数据更快）。
  - 试用后能一键提交反馈与购买意向，并可追踪处理状态。

### 2.2 产品提供方（供给方）

- 核心任务
  - 我想高效上架并配置试用，让潜在客户精准触达；通过需求画像与试用反馈驱动迭代与定价策略。
- 成功标准
  - 从开始录入到完成上架审核提交 ≤ 30 分钟（AI 辅助填充）。
  - 能按行业/场景/预算查看需求聚类与试用转化数据，并导出线索。

### 2.3 平台管理方（运营/审核/技术）

- 核心任务
  - 我想用权限、审核、审计确保合规；用数据与语料闭环持续优化 AI 与运营转化。
- 成功标准
  - 关键数据（访问、对话、推荐、试用、意向）全链路可追溯。
  - 审核流程可配置、可审计；异常行为可告警。

## 3. 三条“闭环主链路”（后续实现的骨架）

### 3.1 购买方闭环：需求 → 推荐 → 试用 → 报告 → 反馈/意向

1. 进入平台（登录/扫码/游客）→ 识别身份与行业（或引导补全）
2. AI 助手发起对话，按“需求收集表”多轮提问
3. 自动生成结构化需求与选型指标（可编辑确认）
4. 从产品库/方案库匹配并推荐 1-3 套方案（可对比）
5. 进入在线试用工作台：开通临时环境/演示租户/试用账号
6. 上传测试数据或选择样例数据 → 体验核心能力 → 生成试用报告
7. 一键提交反馈、问题、评分与购买意向 → 推送给供方并形成工单/线索

### 3.2 供方闭环：上架 → 试用配置 → 线索洞察 → 迭代发布

1. AI 辅助录入产品/能力/场景/案例/价格/版本等信息（生成标签）
2. 配置试用：可用功能、数据范围、权限、样例数据、报告模板
3. 提交审核 → 发布到产品库/方案库
4. 查看需求画像、试用记录、反馈与意向 → 分析转化漏斗
5. 迭代版本/组合方案/价格策略 → 再次提交审核发布

### 3.3 平台闭环：权限/审核/审计 → 数据监控 → AI 优化

1. 组织/角色/权限/数据范围配置
2. 产品与内容审核、试用流程管控
3. 全链路日志与埋点：对话语料、推荐结果、试用行为、反馈与转化
4. 用指标驱动 AI 与推荐算法优化（成功率、满意度、转化率等）

## 4. 核心信息架构（按角色/页面视角）

### 4.1 购买方门户（P0 必做）

- 首页：AI 对话入口、最近推荐、热门场景/方案、试用入口、收藏/最近访问
- 需求中心：我的需求（结构化指标、历史对话、修改/复用）
- 推荐中心：推荐方案列表（1-3 套为主）、方案详情、方案对比
- 试用工作台：环境开通、数据管理、功能体验入口、试用报告
- 反馈与意向：反馈提交、状态追踪、历史记录

### 4.2 供方工作台（P0~P1）

- 产品管理：上架/编辑/上下线、版本管理、标签与能力管理、案例/价格
- 方案管理：组合方案配置、价值说明、试用场景配置
- 试用配置：功能开关、数据范围、试用权限、样例数据、报告模板
- 需求洞察：需求画像聚类、热门场景、试用记录、转化漏斗、反馈列表
- 线索与意向：意向列表、导出、跟进状态（可对接 CRM）

### 4.3 管理后台（P0 必做）

- 组织与权限：组织/用户/角色/菜单权限/数据权限/试用权限
- 审核中心：产品上架审核、方案审核、内容审核、试用配置审核
- 运营看板：访问、对话、推荐、试用、意向、留存等指标
- 日志审计：访问日志、推荐日志、试用日志、修改日志、异常告警
- AI 运营：语料管理、推荐成功率监控、规则/提示词/策略配置（按实际实现方式）

## 5. 功能域拆解（面向实现的“模块化清单”）

### 5.1 AI 交互与需求结构化（P0）

- 对话引导（多轮）：行业/场景/痛点/预算/现状系统/数据基础/数字化成熟度
- 需求结构化：把自然语言转为“可选型指标”与标签（支持用户编辑确认）
- 需求资产化：需求保存、复用、版本化（至少保留历史记录与时间线）
- 纠偏机制：用户反馈“不是这个/不准确”，触发二次提问与指标修正

输出物（对后续模块的输入）：

- 结构化需求（Requirement）
- 选型指标（Criteria）
- 场景标签（Scene Tags）

### 5.2 推荐、方案与对比（P0）

- 推荐：基于结构化需求匹配产品/组合方案，输出 1-3 套为主
- 解释性：给出“为什么推荐”（对应指标命中、风险点、前置条件）
- 方案详情：价值点、适配场景、成本预估、实施周期、案例、架构图（先占位字段）
- 横向对比：能力/成本/周期/风险/前置条件/可试用范围

### 5.3 在线试用工作台（P0）

- 环境开通：临时仿真环境/演示租户/试用账号（与后端实现方式解耦，先定义状态机）
- 数据管理：上传企业测试数据/选择样例数据、数据隔离与权限
- 功能体验：以“方案/产品”为维度展示可试用能力入口（看板/路径优化/多式联运等）
- 试用记录：记录关键操作与结果（最小化可实现：关键步骤打点 + 结果文件/截图上传）
- 试用报告：自动生成（可模板化）：输入数据概览、操作路径、关键结果、结论与建议

### 5.4 反馈、评价与购买意向（P0）

- 一键反馈：评分、问题类型、描述、附件、是否愿意被联系
- 意向表单：预算范围、期望上线时间、联系人信息（需合规提示）
- 状态流转：已提交 → 供方已查看 → 供方已跟进/已回复 → 已关闭
- 推送机制：将反馈与意向推送给对应供方（站内通知/邮件/接口，按实际能力实现）

### 5.5 供方产品/方案全生命周期（P0~P1）

- AI 辅助上架：对话式填充产品字段、自动打标、生成卖点与适配场景
- 版本管理：版本号、更新内容、兼容性、发布时间、回滚策略（先字段化）
- 组合方案：多产品打包、组合逻辑、价值说明、试用配置继承与覆盖
- 上下线与审核：草稿/待审/已上线/已下线/驳回（含驳回原因）

### 5.6 权限、安全、审计（P0）

- 四级管控：组织 / 用户 / 角色 / 权限（菜单 + 数据 + 操作 + 试用）
- 数据隔离：购买方数据与试用数据隔离；供方只能访问授权范围数据
- 日志审计：访问、对话、推荐、试用、修改等关键行为可追溯
- 本地化部署能力（P1）：需求上要保留“离线运行/数据不出域”的架构约束

### 5.7 运营与 AI 持续优化（P0~P1）

- 指标体系：推荐成功率、试用转化率、意向率、留存、供方响应时效
- 语料闭环：对话与反馈沉淀为可用语料（脱敏/合规），驱动策略迭代
- 内容审核流：产品/方案/试用配置/活动等发布审核

## 6. 数据与对象模型（实现前必须统一的“语言”）

最小核心实体（建议优先落地）：

- User：身份信息、所属组织、角色集合、行业、偏好
- Org：组织结构、成员、可见数据范围
- Role/Permission：菜单权限、数据权限、操作权限、试用权限
- Product：能力、场景标签、案例、价格、版本、供方信息、试用配置引用
- Solution（组合方案）：包含多个 Product、组合逻辑、价值说明、试用配置
- Conversation：对话会话、轮次、用户输入、AI 输出、结构化提取结果
- Requirement：结构化需求（场景/指标/预算/成熟度等）
- Recommendation：本次推荐结果（候选列表、排序、解释、置信度）
- Trial：试用实例（关联方案/产品、环境状态、数据集、报告、操作记录）
- Dataset：样例/上传数据元信息、权限与隔离策略
- Feedback：评分、问题、意向、附件、状态流转、供方处理记录
- AuditLog：关键操作日志

## 7. 优先级（为了“先做能跑通闭环”）

### P0（第一阶段必须跑通的最小闭环）

- 购买方：AI 对话采集 → 结构化需求 → 推荐 1-3 套 → 方案对比 → 进入试用 → 生成报告 → 提交反馈/意向
- 供方：产品上架（可先不强依赖 AI 填充）→ 试用配置 → 接收反馈/意向
- 管理方：账号/角色/权限 → 上架审核 → 日志审计 → 基础运营看板

### P1（闭环增强与效率提升）

- 个性化首页与动态布局、收藏/自定义常用模块
- AI 上架提效（更强的字段补全/打标/卖点生成）、组合方案高级配置
- 需求洞察看板（行业/场景/预算维度）与转化漏斗精细化
- 本地化部署与离线运行配套能力
- AI 运营工具（语料管理、策略配置、A/B 实验等）

## 8. 关键验收点（把“功能是否完成”变成可检查项）

- 需求结构化：同一用户输入能稳定产出可编辑的选型指标（不少于：场景、预算、现状系统、数据基础、成熟度）
- 推荐结果：每次输出 1-3 套方案，并给出对应指标命中解释与风险/前置条件
- 试用可用：可创建试用实例并管理数据集；能产出可下载/可查看的试用报告
- 反馈闭环：反馈/意向能关联到具体方案并进入状态流转，供方可查看处理记录
- 权限与审计：不同角色看见的菜单与数据范围正确；关键动作都有审计记录
